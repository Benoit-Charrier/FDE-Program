1
Ingest
Fully automated
Input
  • PDF or DOCX vendor contract
  • Up to 40 pages
  • From "Input Contracts" folder
Agent Task
  • Convert document to structured text
  • Preserve page-level references
  • Detect and report format errors or page-limit breaches
Output
  • Page-segmented text array
  • Each page: number + extracted text
  • IngestionError on failure (graceful)
Human Review

None — fully automated. Errors are logged to audit trail.

2
Extract
Fully automated
Input
  • Page-segmented text from Ingest
Agent Task
  • LLM extracts verbatim text for 7 clause families
  • Assigns confidence score 0–1 per clause
  • Records source page number
liability_cap dpa termination ip_ownership sla governing_law indemnity
Output
  • Clause extraction record per family
  • Verbatim text + source page
  • Confidence score (0.0–1.0)
  • found: true / false flag
Human Review

None — confidence scores feed automatically into the next gate.

3
Confidence Gate
Rule-based — no LLM
Fully automated
Input
  • Clause extraction records
  • Configured threshold (default 0.70)
Agent Task
  • Compare each clause confidence to threshold
  • Flag mandatory clauses below threshold as escalate / low_confidence
  • Flag absent mandatory clauses as escalate / missing_mandatory_clause
  • Passed clauses continue to Classify
Output
  • Pass / fail per clause
  • Failed clauses bypass playbook → go directly to escalation queue
low_confidence (ESC-06) missing_mandatory (ESC-07)
Human Review

None — deterministic rule, not an LLM decision.

4
Classify
Fully automated
Input
  • Clauses that passed the confidence gate
  • Negotiation playbook (JSON config)
Agent Task
  • Compare clause text to accepted positions first
  • If no match, check escalation triggers
  • If no trigger, check negotiable deviations
  • Otherwise escalate with reason no_playbook_entry
  • Assign status + reason code per clause
Output
  • Classification record per clause
match negotiable_deviation escalate no_playbook_entry (ESC-08)

ESC-08 fires when a clause is present and extractable but matches no accepted position, trigger, or deviation — the safe-failure fallback of last resort.

Human Review

None — fully automated. Classifications feed directly into routing.

5
Route
Fully automated
Input
  • Full set of clause classifications for the contract
Agent Task
  • All clauses match → standard queue
  • Any negotiable_deviation, no escalate → playbook_negotiable
  • Any escalate → senior_lawyer_escalation
  • Record per-clause rationale for routing decision
Output
  • Routing decision (one of three queues)
  • Per-clause rationale
Human Review

None per contract — routing is a deterministic rule application, not a recommendation requiring human acceptance.

During calibration: legal ops spot-checks routing on a sample basis. This is a monitoring activity, not a per-contract approval step.

playbook_negotiable path only
6
Redlines
playbook_negotiable contracts only
Fully automated
Input
  • Clauses classified as negotiable_deviation
  • Playbook approved fallback language
Agent Task
  • Substitute approved fallback language verbatim from playbook
  • Cite playbook entry used for each redline
  • If no approved fallback exists: reclassify clause as escalate (never invent language)
Output
  • Draft redline per deviated clause
  • Playbook citation per change
  • Original text preserved alongside proposed text
Human Review
  • Paralegal reviews draft redlines before submission to lawyer
  • Cannot release without lawyer sign-off (enforced downstream)
⚠ Governance control zone — stages 7 and 8 enforce the hard delegation boundary. No outbound package may leave legal without named-lawyer sign-off recorded in the system.
7
Approval Packet
Governance control
Input
  • Draft redlines
  • Source clause text
  • Playbook references
Agent Task
  • Assemble structured approval packet
  • Set release status to BLOCKED immediately
  • Record which clauses require sign-off
Output
  • Approval packet: source clause / proposed redline / playbook citation / approval field
release status: BLOCKED
Human Review
  • Named lawyer reviews each redlined clause
  • Records sign-off with timestamp and identity
  • This is mandatory and non-delegatable
8
Release Gate
Hard system control — no human override possible
Governance control
Input
  • Approval packet
  • Recorded lawyer sign-offs (or absence thereof)
Agent Task
  • Verify every redlined clause has a named-lawyer approval with timestamp
  • Reject release if any sign-off is missing
  • Log any attempted bypass with timestamp + user identity
  • Clear release only when all sign-offs are present
Output
CLEARED — all approved BLOCKED — missing sign-off
  • Audit log entry with outcome
  • Blocked bypass attempts logged with evidence
Human Review

System enforces — no human override possible.

The block is architectural, not advisory. A user cannot override it by claiming approval without the named-lawyer record existing in the system.

9
Report & Audit
Periodic human review
Input
  • All results from all stages
  • Clause extractions, classifications, routing decisions, redlines, approvals
Agent Task
  • Generate HTML report summarising all contracts processed
  • Compute operational metrics (turnaround, route distribution, override rate)
  • Append timestamped audit log entry — append-only, queryable by contract ID
Output
  • HTML report (Output/report.html)
  • audit.jsonl — structured, append-only
  • Metrics: turnaround, route distribution, escalation rate
Human Review
  • Legal ops reviews metrics dashboard periodically
  • Audit trail available for compliance spot-checks
  • No action required per contract — periodic oversight only
